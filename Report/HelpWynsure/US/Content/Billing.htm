Billing

Billing services

A billing service defines the billing properties of the contract coverages. It defines the bill date (frequency, begin of first period, begin of second period), the billed coverages, the customer contact, the invoice number generator, and the distribution type.
Dry periods and premium holidays are also defined in the billing service. Dry periods are used to set a period where the client will not be billed for the coverage; the amount of the period will be spread among the other periods of the year. Premium holidays are used to set a period free of charge.
A billing fee can be set as a percentage of the period premium, or as a flat rate.

Main screens of billing services

 

List billing service

A list billing service can be applied to all members, or members can be split among billing services using classifications. Members can also be added individually to a billing service.

Main screens of list billing services

Summary list billing service

A summary list billing service uses the same screens as a list billing service. It will generate only one line at the group coverage level instead of one line per covered member as in the case of the regular list billing service.

Individual billing service

An individual billing service is set up as a list billing service. When a member is enrolled and falls into the classification of an individual billing service, a copy of the billing service is made for the member. One can customize this copy to adjust it to the member. The billing will create a separate bill for each member.

SA billing

For SA (self administered) billing, the billing service contains an extra folder dedicated to SA enrolment. SA profiles are created automatically using the information from the pricing rule. The volumes and number of lives can be modified using the ‘Edit profiles’ functionality.

The administration of the enrolment can be done either in the billing service (using the ‘Edit profile’ functionality) or in the account viewer. In the account viewer, a user can cancel the last bill, modify the enrolment (and set back charges or credits) and bill the contract.

Online billing

Wynsure provides the online billing functionality. A user can generate the bill for the current or the following period.

Main screens of online billing

Bill Preview

While creating, modifying or consulting a contract, a user can generate a bill preview. This preview shows the bill as if it where generated at the current time.

Main screens of bill preview

Batch billing

The billing batch is used to generate all the bills for a defined period. A date can be set as the end date of the period. All bills scheduled before the end of the period and that have not already been generated will be created.
For further information, refer to the billing batch documentation.

Adjustment management

For each modification that may affect the billing, an adjustment demand is created. These modifications can be:

·         Member modification,

·         Person modification,

·         Contract modification,

·         Delivered service or fee modification.

Each adjustment demand contains the following information:

·         The contract to recalculate,

·         The entity modified,

·         The effective date of the modification.

For each billing, all the new adjustments are taken into account. The contract is recalculated since the oldest modification effective date. If a difference is found between what has already been billed and the calculated amount, appropriate adjustments lines are created.

Main screen for adjustment demand

Logs

For each run of the online or batch billing, Wynsure creates a billing report that gives the following information:

·         The start and end date and time,

·         The number of bills that have been created,

·         The number and list of errors.
